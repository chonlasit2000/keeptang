## 1. แก้ layout หน้า Dashboard และหน้ารายการบนจอคอม (ยังเพี้ยน)

**ปัญหาที่พบบนจอคอม (จอ ≥ 768px):**
- การ์ดรายการแต่ละใบ **สูงไม่เท่ากัน** เรียงเหลื่อมๆ ดูรก
- รายการถูกแบ่งเป็น **2 คอลัมน์ซ้าย-ขวา** ทำให้รายการที่เรียงตามวันถูกหั่นไปคนละฝั่ง อ่านลำดับเวลาไม่รู้เรื่อง

**ที่ต้องการ — ทั้งหน้า Dashboard และหน้ารายการทั้งหมด:**
- รายการ transaction เรียงเป็น **แถวเดียวต่อเนื่องลงมา (timeline เดียว)** เต็มความกว้างของพื้นที่เนื้อหา — ห้ามแตกเป็น 2 คอลัมน์
- จัดกลุ่มตามวันที่ หัววันที่ (เช่น "31 พฤษภาคม 2026") เป็นแถวเดียวเต็มความกว้าง รายการของวันนั้นเรียงอยู่ใต้หัววัน
- การ์ดรายการทุกใบ **ความสูงเท่ากัน** — กำหนดความสูงคงที่หรือใช้ align ให้เนื้อหาในการ์ดไม่ดันให้สูงต่ำไม่เท่ากัน
- การ์ดรายการขยายเต็มความกว้าง container (ไม่ใช่การ์ดแคบๆ ลอยกลาง)
- คงความกว้างเนื้อหาสูงสุดไว้ (เช่น `max-w-5xl` จัดกึ่งกลาง) เพื่อไม่ให้ยืดเต็มจอกว้างจนอ่านยาก

**สิ่งที่ทำถูกแล้ว — คงไว้:** sidebar ซ้าย, การ์ดสรุป 3 ใบเรียงแนวนอน, ปุ่มเพิ่มรายการใน sidebar
**คงสไตล์ B:** สีครีม/coral/พาสเทล, มุมโค้ง, ฟอนต์ — เปลี่ยนแค่การจัดวาง

**จอมือถือ (< 768px):** คงคอลัมน์เดียว + bottom nav เหมือนเดิม ไม่ต้องแก้

## 2. ทำหน้าเพิ่ม/แก้หมวดหมู่ใหม่ — เลือกไอคอนและสีแบบเห็นภาพ

**ปัญหา:** ตอนนี้หน้าเพิ่มหมวดหมู่ให้เลือกไอคอนและสีผ่าน dropdown ที่เป็น **ข้อความภาษาอังกฤษ** ("Circle", "coral") ผู้ใช้มองไม่เห็นว่าไอคอนหน้าตายังไง สีอะไร ต้องเดา — ใช้งานยากมาก

**ที่ต้องการ — เปลี่ยน dropdown ข้อความเป็นตัวเลือกแบบเห็นภาพ:**

- **เลือกไอคอน:** แสดงเป็น **ตารางไอคอน (grid)** ไอคอนจริงให้กดเลือก แตะไอคอนไหนไอคอนนั้นถูกเลือก (มีกรอบ/ไฮไลต์บอกว่าเลือกอยู่) — เตรียมไอคอนให้เลือกสัก 20–30 แบบที่เหมาะกับหมวดรายรับรายจ่าย (อาหาร เดินทาง ช้อปปิ้ง บิล สุขภาพ บันเทิง เงินเดือน ฯลฯ)
- **เลือกสี:** แสดงเป็น **วงกลมสีจริง (swatch)** เรียงให้กดเลือก แตะสีไหนสีนั้นถูกเลือก (มีกรอบบอกว่าเลือกอยู่) — ใช้ชุดสีพาสเทลที่เข้ากับสไตล์ B
- **ตัวอย่างสด:** ชิปไอคอนตัวอย่างด้านบนต้องอัปเดตทันทีเมื่อเปลี่ยนไอคอนหรือสี
- ฟิลด์ชื่อหมวดหมู่ และตัวเลือกกลุ่ม (`grp`: จำเป็น/ฟุ่มเฟือย/ออม-ลงทุน/รางวัลตัวเอง) คงไว้เหมือนเดิม
- ใช้หน้าตา/ตัวเลือกชุดเดียวกันนี้ทั้งตอน "เพิ่ม" และตอน "แก้ไข" หมวดหมู่

**หมายเหตุ:** ข้อมูล `icon` และ `color` มีคอลัมน์อยู่ในตาราง `categories` แล้ว — งานนี้แค่เปลี่ยนวิธีให้ผู้ใช้เลือก ไม่ต้องแก้ schema

## 3. เก็บงานเทสต์มือถือ / PWA ที่ยังค้าง

ส่วนนี้ยังไม่ได้ทดสอบใน Phase 1 — ให้ Codex ตรวจและแก้ให้ผ่านทุกข้อ:

- **เปิดบนจอมือถือจริง (หรือย่อหน้าต่างเบราว์เซอร์ให้แคบ ~375px):**
  - ทุกหน้า (Dashboard, เพิ่มรายการ, รายการทั้งหมด, ตั้งค่า, Login) ต้องแสดงผลครบ ไม่มีของล้นจอ ไม่มีต้องเลื่อนแนวนอน
  - ปุ่มและพื้นที่กดต้องใหญ่พอกดด้วยนิ้ว (อย่างน้อย ~44px)
  - แป้นตัวเลขในหน้าเพิ่มรายการกดสะดวกบนมือถือ
  - bottom nav แสดงถูกต้อง ไม่ทับเนื้อหา
- **PWA — ติดตั้งได้:**
  - มี manifest ครบ (ชื่อแอป "keeptang", ไอคอนหลายขนาด, theme color, background color)
  - มี service worker ทำงาน
  - เปิดบนมือถือแล้วสามารถ "เพิ่มลงหน้าจอหลัก" / "ติดตั้งแอป" ได้
  - เปิดจากไอคอนที่ติดตั้งแล้วแสดงแบบเต็มจอ (standalone) ไม่มีแถบ URL ของเบราว์เซอร์
- ไม่ต้องทำให้ใช้งาน offline เต็มรูปแบบ (เก็บไว้ Phase หลัง) — แค่ติดตั้งเป็น PWA ได้ก็พอ
