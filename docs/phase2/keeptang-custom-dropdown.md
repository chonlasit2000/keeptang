# keeptang — สร้าง custom dropdown component แทน `<select>` ทั่วทั้งแอป

> สเปกงานสำหรับ Codex — งานออกแบบ component ใหม่
> ทำหลังจาก `keeptang-fix-chart-polish.md` ผ่านแล้ว
> คัดลอกไฟล์นี้ไปวางใน Codex หรือวางไฟล์ไว้ในโฟลเดอร์โปรเจกต์แล้วสั่งให้อ่าน

**สำคัญ:** งานนี้แตะหลายหน้าจอ — ห้ามกระทบฟังก์ชันเดิม (ค่าที่บันทึก/กรอง/แสดงต้องเหมือนเดิมทุกอย่าง) แค่เปลี่ยนหน้าตาของตัวเลือก

---

## ที่มา

ตอนนี้แอปใช้ `<select>` ของเบราว์เซอร์ในหลายจุด ซึ่งหน้าตาต่างกันตามอุปกรณ์ ตามภาษา และตามเบราว์เซอร์ (เช่นเดียวกับ `<input type="date">` ที่เปลี่ยนไปแล้ว) คุม UI ให้เข้ากับสไตล์ B ไม่ได้

แก้โดยสร้าง custom dropdown component ของเราเอง ใช้ซ้ำได้ทุกจุดในแอป

---

## สร้าง component ที่ใช้ซ้ำได้

สร้าง component ใหม่ (เช่น `components/Select.jsx` หรือชื่อที่เหมาะกับโครงสร้างเดิม) ที่ใช้แทน `<select>` ได้ในทุกจุด

### พฤติกรรม
- คลิกที่ช่อง → ดรอปดาวน์เปิด แสดงรายการตัวเลือก
- คลิกตัวเลือก → ช่องอัปเดต ดรอปดาวน์ปิด
- คลิกนอกพื้นที่ดรอปดาวน์ → ปิด
- กด ESC → ปิด
- เลื่อน scroll ในรายการได้ถ้ามีตัวเลือกเยอะ
- รองรับ keyboard: ลูกศรขึ้น/ลง, Enter เลือก
- รองรับ controlled component pattern (รับ `value` และ `onChange` แบบเดียวกับ `<select>` เพื่อแทนที่ง่าย)

### ดีไซน์ — สไตล์ B
- ช่อง dropdown ปิด: พื้นขาว มุมโค้ง (เหมือนการ์ดอื่นๆ ในแอป) มีไอคอน chevron ขวา
- พาเนลตัวเลือกเมื่อเปิด: พื้นขาว เงานุ่ม มุมโค้ง อยู่ใต้ช่อง
- ตัวเลือกที่เลือกอยู่: ไฮไลต์ด้วยสีพื้น coral อ่อนๆ
- ตัวเลือกที่ hover: พื้นครีมอ่อน
- ฟอนต์เดียวกับแอป (Noto Sans Thai / IBM Plex Sans Thai)

### รองรับการแสดงไอคอน + สีในตัวเลือก (สำคัญ)
- แต่ละตัวเลือกสามารถมี: ไอคอน (lucide-react) + สี + ข้อความ
- ใช้ในตัวกรองหมวดหมู่ — แต่ละหมวดต้องแสดงไอคอนและสีของหมวดนั้น
- ถ้า prop ไอคอน/สีไม่ได้ส่งมา ให้แสดงแค่ข้อความปกติ

### responsive
- บนจอคอม: dropdown ปกติเปิดลง
- บนมือถือ: dropdown ใช้งานง่ายด้วยนิ้ว — ตัวเลือกแต่ละอันสูงพอกด (~44px), ถ้ามีตัวเลือกเยอะให้เลื่อนได้

## จุดที่ต้องแทนที่ในแอป

ไล่หาทุก `<select>` ในโค้ดเดิม แล้วเปลี่ยนเป็น component ใหม่ ที่พบแน่ๆ:

1. **ตัวกรองหมวดหมู่หน้ารายการทั้งหมด** — ปัจจุบันเป็น dropdown หมวด ให้ใช้ component ใหม่และโชว์ไอคอน+สีของแต่ละหมวด
2. **ดรอปดาวน์เลือกกลุ่ม (จำเป็น/ฟุ่มเฟือย/ออม-ลงทุน/รางวัลตัวเอง)** ในหน้าเพิ่ม/แก้หมวดหมู่ — ไม่ต้องใช้ไอคอน แต่หน้าตาเข้ากับสไตล์
3. **จุดอื่นๆ** ที่ยังใช้ `<select>` ของเบราว์เซอร์อยู่ — ไล่หาให้ครบ

**ห้ามแตะ:**
- ตารางเลือกไอคอนและวงเลือกสี ในหน้าเพิ่มหมวดหมู่ — สองอันนี้ไม่ใช่ dropdown ทำดีอยู่แล้ว
- ปฏิทินวันที่ (react-day-picker) — เป็น component อื่น

---

## ทดสอบ

- ทุก dropdown ในแอปเปลี่ยนเป็น component ใหม่ หน้าตาเหมือนกันทุกจุด
- ตัวกรองหมวดหมู่ในหน้ารายการ: แต่ละหมวดแสดงไอคอน+สี กรองได้ถูกต้องเหมือนเดิม
- dropdown กลุ่มในหน้าเพิ่ม/แก้หมวดหมู่: เลือกได้ บันทึกค่าถูก
- ทดสอบบนมือถือ: เปิด/เลือก/ปิด ทำได้สะดวกด้วยนิ้ว
- ทดสอบบน iOS Safari: ไม่มีปัญหาล้นกรอบหรือ style แปลก
- คลิกนอกพื้นที่ → ปิด, กด ESC → ปิด, ใช้ keyboard ได้
- ไม่มี `<select>` ของเบราว์เซอร์เหลือในแอป
- ค่าที่บันทึกลง database ถูกต้องเหมือนเดิม (ตรวจใน Supabase)
- ฟังก์ชันเดิมทั้งหมดทำงานครบ
