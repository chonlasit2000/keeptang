## เป้าหมาย Phase 2

เพิ่มหน้า **"สถิติ"** ที่เปลี่ยนข้อมูลรายรับรายจ่ายที่บันทึกไว้ให้เห็นเป็นภาพ ช่วยให้ผู้ใช้เข้าใจว่าเงินไปไหนบ้าง

**ขอบเขต Phase 2 นี้:** กราฟโดนัทแยกหมวด + กราฟแนวโน้มรายเดือน + สรุป 4 กลุ่มรายจ่าย
**ยังไม่ทำใน Phase 2:** ผัง Flow (Sankey diagram) — ยกไป Phase 3

**สำคัญ:** ห้ามกระทบฟังก์ชันและหน้าจอเดิมที่ทำงานได้แล้วใน Phase 1 — งานนี้เป็นการ "เพิ่ม" หน้าใหม่เป็นหลัก

## Tech stack เพิ่มเติม

- ใช้ **recharts** สำหรับวาดกราฟ (โดนัทและกราฟเส้น/แท่ง) — เข้ากับ React ที่ใช้อยู่
- ไม่ต้องเพิ่ม dependency อื่นนอกจากนี้
- ดึงข้อมูลจาก Supabase ด้วย hook เดิม (`useTransactions`, `useCategories`) — ถ้าต้องเพิ่ม query ใหม่ ให้ทำในรูปแบบเดียวกับของเดิม

## หน้าใหม่: หน้าสถิติ

เพิ่มเมนู **"สถิติ"** เข้าไปใน navigation (ทั้ง bottom nav บนมือถือ และ sidebar บนจอคอม)

หน้าสถิติเป็น **หน้าเดียวเลื่อนยาว** ประกอบด้วย 4 ส่วนเรียงจากบนลงล่าง:

### ส่วนที่ 0 — ตัวเลือกเดือน
- ด้านบนสุดของหน้า มีตัวเลือกเดือน (เลื่อนเดือนก่อนหน้า/ถัดไปได้) แบบเดียวกับหน้า Dashboard
- ทุกกราฟในหน้านี้อิงเดือนที่เลือก เปลี่ยนเดือนแล้วทุกกราฟอัปเดตตาม

### ส่วนที่ 1 — กราฟโดนัทแยกหมวด (รายจ่าย)
- กราฟโดนัท (donut/pie) แสดงสัดส่วนรายจ่ายของเดือนที่เลือก แบ่งตามหมวดหมู่
- แต่ละชิ้นใช้สีของหมวดนั้น (จากฟิลด์ `color`)
- มี legend บอกชื่อหมวด + จำนวนเงิน + เปอร์เซ็นต์
- ตรงกลางโดนัทแสดงยอดรายจ่ายรวมของเดือน
- เรียงหมวดจากมากไปน้อย
- ถ้าเดือนนั้นไม่มีรายจ่าย แสดง empty state ("ยังไม่มีรายจ่ายในเดือนนี้")

### ส่วนที่ 2 — กราฟแนวโน้มรายเดือน
- กราฟแสดงรายรับและรายจ่ายรวมของแต่ละเดือน ย้อนหลัง 6 เดือน (รวมเดือนปัจจุบัน)
- ใช้กราฟแท่งคู่ (รายรับ/รายจ่าย แท่งคนละสี) หรือกราฟเส้น 2 เส้น — เลือกแบบที่อ่านง่าย
- สีรายรับเขียว สีรายจ่ายชมพู/แดง ตามโทนสไตล์ B
- มีแกนบอกเดือน และค่าเงิน
- แตะ/ชี้ที่แท่งหรือจุด แสดงตัวเลขที่แน่นอน (tooltip)

### ส่วนที่ 3 — สรุป 4 กลุ่มรายจ่าย
- แบ่งรายจ่ายของเดือนที่เลือกตามฟิลด์ `grp` ของหมวดหมู่ ออกเป็น 4 กลุ่ม:
  - จำเป็น (need)
  - ฟุ่มเฟือย (want)
  - ออม/ลงทุน (saving)
  - ให้รางวัลตัวเอง (reward)
- แสดงเป็น **การ์ด 4 ใบ หรือแถบเปอร์เซ็นต์ (bar)** — แต่ละกลุ่มบอกจำนวนเงินรวม + เปอร์เซ็นต์เทียบกับรายจ่ายทั้งหมด
- แต่ละกลุ่มมีสีประจำกลุ่มที่ต่างกันชัดเจน
- ยังไม่ต้องทำผัง Flow — แค่สรุปตัวเลข 4 กลุ่มแบบอ่านง่ายก็พอ (ผัง Flow เก็บไว้ Phase 3)
- ถ้ากลุ่มไหนไม่มีข้อมูล แสดงเป็น 0 / 0% ไม่ใช่ซ่อนทิ้ง

---

## การจัดวาง (responsive)

- **จอมือถือ (< 768px):** ทุกส่วนเรียงเป็นคอลัมน์เดียว เลื่อนลงดู
- **จอคอม (≥ 768px):** ใช้พื้นที่ให้คุ้ม — กราฟโดนัทกับกราฟแนวโน้มอาจวาง 2 คอลัมน์ในแถวเดียวกันได้ ส่วน 4 กลุ่มเรียง 4 ใบในแถวเดียว ใช้ layout ที่อ่านสบาย ไม่อัดแน่น
- คงสไตล์ B: สีครีม/coral/พาสเทล, การ์ดมุมโค้ง, ฟอนต์เดิม
- กราฟทุกตัวต้องไม่ล้นกรอบบนจอแคบ

## Acceptance criteria

- [ ] มีเมนู "สถิติ" ใน navigation ทั้งมือถือและจอคอม
- [ ] กราฟโดนัทแสดงสัดส่วนรายจ่ายตามหมวด ใช้สีหมวดถูกต้อง มี legend + เปอร์เซ็นต์
- [ ] กราฟแนวโน้มแสดงรายรับ/รายจ่ายย้อนหลัง 6 เดือน มี tooltip
- [ ] สรุป 4 กลุ่มรายจ่ายคำนวณจาก `grp` ถูกต้อง รวมเปอร์เซ็นต์ได้ 100%
- [ ] เปลี่ยนเดือนแล้วทุกกราฟอัปเดตตาม
- [ ] กรณีไม่มีข้อมูล แสดง empty state ไม่ใช่จอว่างหรือ error
- [ ] ตัวเลขเงินทุกที่แสดงรูปแบบ `฿1,234`
- [ ] responsive ดีทั้งมือถือและจอคอม กราฟไม่ล้นกรอบ
- [ ] ฟังก์ชันและหน้าจอเดิมจาก Phase 1 ยังทำงานครบ

## ลำดับงานที่แนะนำ

1. ติดตั้ง recharts + เพิ่มเมนู "สถิติ" ใน navigation + สร้างหน้าเปล่า + ตัวเลือกเดือน
2. ส่วนที่ 1 — กราฟโดนัทแยกหมวด
3. ส่วนที่ 2 — กราฟแนวโน้มรายเดือน
4. ส่วนที่ 3 — สรุป 4 กลุ่มรายจ่าย
5. เก็บงาน responsive + empty state ทุกส่วน
6. ทดสอบ: เปลี่ยนเดือน, เดือนที่ไม่มีข้อมูล, ย่อ/ขยายจอ, มือถือจริง
7. ยืนยันว่าหน้าจอเดิมทั้งหมดยังทำงานปกติ
