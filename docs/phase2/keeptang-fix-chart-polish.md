# keeptang — แก้บั๊กเล็กในกราฟ

> สเปกงานแก้ไขสำหรับ Codex — บั๊กเล็ก 2 จุดในกราฟ ทำก่อน custom dropdown
> คัดลอกไฟล์นี้ไปวางใน Codex หรือวางไฟล์ไว้ในโฟลเดอร์โปรเจกต์แล้วสั่งให้อ่าน

**สำคัญ:** แก้เฉพาะกราฟทั้ง 2 ตัว (โดนัทและแนวโน้ม) — ห้ามกระทบส่วนอื่นของแอป

---

## 1. ปิดกรอบ focus outline บนกราฟตอนคลิก

**ปัญหา:** เมื่อคลิกที่ส่วนของกราฟ (โดนัทหรือแนวโน้ม) จะมีกรอบดำขึ้นรอบส่วนที่คลิก (focus outline ของเบราว์เซอร์) ดูไม่สวย

**ที่ต้องการ:**
- ปิด focus outline บน **element ของกราฟ Recharts เท่านั้น**
- ใช้ CSS แบบเจาะจง เช่น `.recharts-wrapper *:focus { outline: none; }` หรือ target element ของ Recharts โดยตรง
- **ห้ามปิด focus outline ทั่วทั้งเว็บ** — เพราะ outline บน input/button/link สำคัญสำหรับคนที่ใช้คีย์บอร์ดเปิดเว็บ
- ลองคลิกที่ส่วนต่างๆ ของกราฟ → ไม่มีกรอบขึ้น
- ลองกด Tab ในฟอร์ม (login, เพิ่มรายการ) → ยังเห็น outline บอกว่ากำลัง focus ช่องไหนอยู่ (ไม่หาย)

---

## 2. แก้ tooltip ของกราฟให้ตามเคอร์เซอร์ — แต่ไม่ทับเลขกลางวงโดนัท

**ปัญหา:** ตอนนี้ tooltip ของกราฟลอยอยู่ที่ตำแหน่งคงที่ (มุมซ้ายบน) ไม่ขยับตามจุดที่ hover — เกิดเพราะรอบก่อนแก้ปัญหา tooltip ทับเลขกลางวงด้วยการตั้ง position คงที่ ซึ่งแก้เกินไป

**ที่ต้องการ:**
- tooltip ลอยอยู่ใกล้จุดที่ hover (เคลื่อนตามเคอร์เซอร์)
- **แต่ห้ามทับเลขกลางวงโดนัท** (`฿xxxx`)
- แนวทางที่แนะนำ:
  - ลบ `position` ที่ตั้งคงที่ใน `<Tooltip>` ของ Recharts ออก ให้มันกลับมา default
  - ใช้ `offset` เพื่อให้ tooltip ห่างจากเคอร์เซอร์เล็กน้อย ไม่ลอยทับ
  - ถ้า tooltip ขนาดเล็ก โอกาสทับเลขกลางวงน้อยอยู่แล้ว เพราะมันลอยข้างจุด hover ไม่ใช่ตรงกลาง
  - ถ้าจำเป็น ใช้ `wrapperStyle` กำหนด pointer-events และ z-index ที่เหมาะสม
- เนื้อหา tooltip คงเดิม: ชื่อหมวด + จำนวนเงิน + เปอร์เซ็นต์

---

## ทดสอบ

- คลิกที่ส่วนของกราฟโดนัทและแท่งกราฟแนวโน้ม → ไม่มีกรอบดำ
- กด Tab ในหน้า login และหน้าเพิ่มรายการ → ยังเห็น focus outline ปกติ
- hover บนส่วนต่างๆ ของกราฟโดนัท → tooltip ลอยใกล้จุด hover ไม่ใช่อยู่มุมเดียวตลอด
- เลขกลางวงโดนัท (`฿xxxx`) ยังอ่านออก ไม่ถูก tooltip ทับ
- ฟังก์ชันอื่นไม่กระทบ
