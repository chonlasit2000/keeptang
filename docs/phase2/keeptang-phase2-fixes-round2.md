# keeptang — แก้หน้าสถิติ รอบ 2

> สเปกงานแก้ไขสำหรับ Codex — เก็บงานหน้าสถิติที่ยังเพี้ยนหลังแก้รอบแรก
> คัดลอกไฟล์นี้ไปวางใน Codex หรือวางไฟล์ไว้ในโฟลเดอร์โปรเจกต์แล้วสั่งให้อ่าน

**สำคัญ:**
- ห้ามกระทบฟังก์ชันเดิมและส่วนที่ทำถูกแล้ว — โดยเฉพาะการคำนวณข้อมูล 4 กลุ่ม, กราฟโดนัท, การเปลี่ยนเดือน
- งานนี้เน้นบั๊กการแสดงผล ไม่ใช่ logic ข้อมูล — อย่ารื้อ logic การดึงข้อมูลเดิม

---

## 1. 🔴 ด่วน: กราฟแนวโน้ม 6 เดือนหายไปทั้งคอมและมือถือ

**ปัญหา:** หลังแก้รอบแรก การ์ด "แนวโน้ม 6 เดือน" **ว่างเปล่า** — มีแค่หัวข้อกับคำบรรยาย ไม่มีกราฟแสดงเลย (ทั้งจอคอมและมือถือ)

**ที่ต้องการ:**
- กราฟแท่งคู่ (รายรับ/รายจ่าย ย้อนหลัง 6 เดือน) ต้องกลับมาแสดง
- ตรวจว่ามีข้อมูลส่งเข้า component กราฟ และ component render ได้จริง
- ตรวจ `<ResponsiveContainer width="100%" height={...}>` — ถ้า height ของ container ภายนอกเป็น auto/ไม่กำหนด ResponsiveContainer จะคำนวณ height ไม่ได้และไม่ render กราฟ ต้องกำหนด height ของ ResponsiveContainer ชัดเจน (เช่น `height={280}`) หรือกำหนด min-height ของ container แม่
- ตรวจ console ของเบราว์เซอร์ว่ามี error/warning จาก Recharts หรือไม่ และแก้ที่ต้นเหตุ
- กราฟกลับมาแล้ว ค่อยกลับไปทำงานเดิมจากรอบแรก: แกน X แสดงครบ 6 เดือน, แกน Y ไม่ซ้ำ, แท่งกระจายเต็มความกว้าง

## 2. โดนัทบนจอคอม — legend ถูกตัด

**ปัญหา:** บนจอคอม ข้อความใน legend ของโดนัทถูกตัด เช่น "เงินออม/ลงทุน" กลายเป็น "เงินออม/..." ทั้งที่การ์ดมีพื้นที่เหลือ

**ที่ต้องการ:**
- ให้ legend แสดงชื่อหมวดเต็ม ไม่ถูกตัด
- ปรับสัดส่วนพื้นที่ระหว่างวงโดนัทและ legend ในการ์ด — อาจลดขนาดวงโดนัทเล็กน้อย หรือเพิ่มความกว้างพื้นที่ legend
- ถ้าจอแคบจริงๆ ค่อยใช้ ellipsis แต่บนจอคอมที่การ์ดกว้างพอ ห้ามตัดข้อความ
- รายการ legend ต้องเห็นครบทุกหมวด (ตอนนี้ดูเหมือนจะมองเห็นเฉพาะ 3 หมวดแรก ต้องตรวจว่าหมวดอื่นๆ ถูกซ่อนหรือเปล่า)

## 3. โดนัท — tooltip ทับตัวเลขกลางวง

**ปัญหา:** เมื่อ hover ที่ส่วนหนึ่งของวงโดนัท tooltip popup ลอยมาทับตัวเลขกลางวง (`฿1,135`) ทำให้อ่านไม่ออก

**ที่ต้องการ:**
- ตำแหน่ง tooltip ไม่ทับตัวเลขกลางวง — ให้ลอยอยู่นอกวง หรือมุมบน/ด้านข้างของกราฟ
- ตั้งค่า Recharts Tooltip ให้ใช้ `position` คงที่ หรือ offset ออกไปจากจุด hover
- เนื้อหา tooltip: ชื่อหมวด + จำนวนเงิน + เปอร์เซ็นต์ ก็พอ

## 4. Empty state — กล่อง "ยังไม่มีรายจ่ายในเดือนนี้" ติดด้านบนการ์ด

**ปัญหา:** เมื่อเดือนที่เลือกไม่มีข้อมูล กล่องข้อความ "ยังไม่มีรายจ่ายในเดือนนี้" ติดอยู่ด้านบนของการ์ดโดนัท ทำให้พื้นที่ด้านล่างโล่งและดูแหวก

**ที่ต้องการ:**
- กล่อง empty state อยู่ **กึ่งกลางการ์ดทั้งแนวตั้งและแนวนอน**
- ใช้ flexbox กับ `justify-content: center; align-items: center;` บนพื้นที่เนื้อหาของการ์ด
- ความสูงของการ์ด empty state ควรใกล้เคียงการ์ดที่มีข้อมูล (ไม่ยุบลีบ)
- ใช้แนวเดียวกันกับทั้งกราฟโดนัทและกราฟแนวโน้ม

## 5. การ์ดสรุป 4 กลุ่ม — หัวข้อกับตัวเลขห่างเกินไป

**ปัญหา:** ในการ์ด 4 กลุ่ม ระหว่างชื่อกลุ่ม ("จำเป็น"/"ฟุ่มเฟือย"/...) กับจำนวนเงิน (`฿295` ฯลฯ) มีช่องว่างมากเกินไป ดูเหมือนถูกอัด space ตรงกลางเพื่อให้สูงเท่ากัน — เกิดทั้งจอคอมและมือถือ

**สาเหตุที่น่าจะเป็น:** ใช้ `justify-content: space-between` กับการ์ดที่สูงเกินจำเป็น เลยดันชื่อขึ้นบนสุด ตัวเลขลงล่างสุด

**ที่ต้องการ:**
- ลดความสูงรวมของการ์ด หรือจัดกลุ่ม "ชื่อ + จำนวนเงิน" ให้อยู่ติดกันด้วย gap ปกติ (เช่น 4–8px)
- โครงสร้างที่แนะนำ: ส่วนหัว (ชื่อกลุ่ม + ชิป %) → จำนวนเงิน (อยู่ใต้ชื่อ ไม่เว้นห่าง) → แถบ progress ที่ก้นการ์ด
- การ์ดทั้ง 4 ใบยังต้องสูงเท่ากัน — แต่ใช้ min-height ของ container แทนที่จะปล่อยให้ space-between ดันเนื้อหา
- ทำให้การ์ดดูกะทัดรัด ไม่โปร่งเกินไป

## ทดสอบหลังแก้

- กราฟแนวโน้มกลับมาแสดงผลทั้งคอมและมือถือ พร้อมแกน X 6 เดือน, แกน Y ไม่ซ้ำ
- โดนัท: legend แสดงชื่อหมวดเต็มทุกหมวด, tooltip ไม่ทับเลขกลางวง
- Empty state อยู่กึ่งกลางการ์ดทั้งกราฟโดนัทและกราฟแนวโน้ม
- การ์ด 4 กลุ่ม: ชื่อกับตัวเลขอยู่ติดกัน ไม่ห่าง, สูงเท่ากันทุกใบ ทั้งคอมและมือถือ
- ไม่มี error ใน console ของเบราว์เซอร์
- ฟังก์ชันและการคำนวณข้อมูลเดิมยังถูกต้องครบ
