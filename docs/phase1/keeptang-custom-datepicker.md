# keeptang — เปลี่ยนช่องเลือกวันที่เป็น date picker แบบกำหนดเอง

> สเปกงานสำหรับ Codex — แทนไฟล์ `keeptang-fix-ios-date.md` เดิม
> คัดลอกไฟล์นี้ไปวางใน Codex หรือวางไฟล์ไว้ในโฟลเดอร์โปรเจกต์แล้วสั่งให้อ่าน

**สำคัญ:** ห้ามกระทบฟังก์ชันเดิม — โดยเฉพาะการบันทึก/แก้ไขรายการ และค่าวันที่ที่ส่งเข้า database

---

## ที่มา

ตอนนี้ช่องเลือกวันที่ใช้ `<input type="date">` ของเบราว์เซอร์ ซึ่งมี 2 ปัญหา:
1. บน iOS Safari กล่อง input ล้นออกนอกกรอบการ์ด
2. ปฏิทินป๊อปอัปเป็นของเบราว์เซอร์ ปรับ UI ให้เข้ากับสไตล์แอปไม่ได้ และหน้าตา/รูปแบบปี (ค.ศ./พ.ศ.) ต่างกันไปตามเบราว์เซอร์และอุปกรณ์

แก้โดยเปลี่ยนไปใช้ date picker ที่วาดด้วยโค้ดเราเอง — คุม UI ได้เต็มที่ หน้าตาเหมือนกันทุกเบราว์เซอร์ และปัญหา iOS หายไปในตัว (เพราะเลิกใช้ native date input)

---

## สิ่งที่ต้องทำ

### 1. ติดตั้งและใช้ react-day-picker
- ใช้ library **react-day-picker** สำหรับวาดปฏิทิน (ทำงานร่วมกับ date-fns ที่โปรเจกต์ใช้อยู่แล้ว)
- แทนที่ `<input type="date">` ทุกจุดที่ใช้เลือกวันที่ (หน้าเพิ่มรายการ และหน้าแก้ไขรายการ)

### 2. รูปแบบการแสดงผล
- **ช่องวันที่:** แสดงเป็นปุ่ม/ช่องที่อ่านได้ เช่น `19 พฤษภาคม 2569` — เดือนเป็นภาษาไทย ปีเป็น **พ.ศ.**
- **ปฏิทิน:** เป็น **ป๊อปอัป** — กดที่ช่องวันที่แล้วปฏิทินเด้งขึ้นมา เลือกวันแล้วป๊อปอัปปิด
- ในปฏิทิน: ชื่อเดือนและชื่อวันเป็นภาษาไทย, ปีหัวปฏิทินแสดงเป็น **พ.ศ.**
- ค่าเริ่มต้นเมื่อเปิดหน้าเพิ่มรายการ = วันปัจจุบัน

### 3. สำคัญมาก — การแปลงปี พ.ศ. / ค.ศ.
- **พ.ศ. ใช้แสดงผลให้ผู้ใช้เห็นเท่านั้น** (พ.ศ. = ค.ศ. + 543)
- ค่าที่เก็บลง database (คอลัมน์ `txn_date`) ต้องเป็น **ค.ศ.** ตามเดิม รูปแบบ `YYYY-MM-DD` — ห้ามเก็บเป็น พ.ศ.
- ตรวจให้แน่ใจว่า logic การคำนวณยอดรายเดือน การกรองตามเดือน และการแสดงรายการ ยังอ้างอิงค่า ค.ศ. ที่ถูกต้อง ไม่เพี้ยนจากการแปลงปี
- ระวังเรื่อง timezone — วันที่ที่เลือกต้องไม่เลื่อนไป 1 วันจากการแปลง (ใช้ local date ไม่ใช่ UTC shift)

### 4. ดีไซน์ — สไตล์ B
- ปฏิทินป๊อปอัปทำให้เข้ากับสไตล์ B: พื้นการ์ดขาว มุมโค้ง เงานุ่ม
- วันที่เลือกอยู่ไฮไลต์ด้วยสีหลัก coral `#D85A30`
- วันปัจจุบันมีจุด/กรอบบอกแบบบางๆ
- ฟอนต์เดียวกับแอป (IBM Plex Sans Thai / Noto Sans Thai)
- ปุ่มเลื่อนเดือนก่อนหน้า/ถัดไป กดง่าย

### 5. responsive
- จอมือถือ: ป๊อปอัปปฏิทินต้องอยู่ในกรอบหน้าจอ ไม่ล้นซ้าย-ขวา กดเลือกวันด้วยนิ้วได้สะดวก (พื้นที่แตะแต่ละวันใหญ่พอ)
- จอคอม: ป๊อปอัปแสดงใกล้ช่องวันที่ ขนาดเหมาะสม
- ทดสอบบน iPhone Safari จริง — ต้องไม่มีการล้นกรอบแบบ input เดิม

---

## ทดสอบ

- เปิดหน้าเพิ่มรายการบน iPhone Safari จริง → ช่องวันที่และปฏิทินอยู่ในกรอบ ไม่ล้น
- เปิดบน Chrome เดสก์ท็อปและ Android → ปฏิทินหน้าตาเหมือนกัน เป็นสไตล์ B
- ปฏิทินแสดงเดือน/ปีเป็นภาษาไทยและ พ.ศ.
- เลือกวันที่ → ช่องอัปเดตเป็นวันที่ที่เลือก (พ.ศ.)
- บันทึกรายการ → ไปตรวจใน Supabase ว่า `txn_date` เก็บเป็น ค.ศ. และเป็นวันที่ถูกต้อง (ไม่เลื่อนวัน)
- โหมดแก้ไข: เปิดรายการเดิม → ปฏิทินแสดงวันที่เดิมถูกต้อง
- ยอดรวมรายเดือนและการกรองเดือนยังถูกต้อง
- ฟังก์ชันเดิมทั้งหมดทำงานครบ
