# keeptang — รายการแก้ไข Phase 1 รอบ 3 (fix-list)

> สเปกงานแก้ไขรอบ 3 สำหรับ Codex — เก็บรายละเอียดที่ยังเหลือจากรอบ 2
> คัดลอกไฟล์นี้ไปวางใน Codex หรือวางไฟล์ไว้ในโฟลเดอร์โปรเจกต์แล้วสั่งให้อ่าน

**สำคัญ:** ห้ามกระทบฟังก์ชันและส่วนที่ทำถูกแล้ว — โดยเฉพาะตารางเลือกไอคอนและวงเลือกสีในหน้าหมวดหมู่ (ทำดีแล้ว คงไว้) แก้เฉพาะ 2 จุดด้านล่าง

---

## 1. แก้การ์ดสรุป 3 ใบบน Dashboard ให้สมดุล

**ปัญหา:** การ์ดสรุป 3 ใบ (รายรับรวม / รายจ่ายรวม / เงินคงเหลือ) ขนาดตัวเลขไม่เท่ากัน — ใบ "เงินคงเหลือ" ตัวเลขใหญ่กว่าอีก 2 ใบมาก ทำให้ 3 ใบดูไม่เป็นชุดเดียวกัน ไม่สมดุล

**ที่ต้องการ:**
- ขนาดฟอนต์ตัวเลขของทั้ง 3 ใบ **เท่ากัน**
- หัวข้อ (รายรับรวม/รายจ่ายรวม/เงินคงเหลือ) ขนาดเท่ากันทั้ง 3 ใบ
- ความสูงการ์ดทั้ง 3 ใบเท่ากัน
- ถ้าต้องการเน้นใบ "เงินคงเหลือ" ว่าเป็นตัวเลขสำคัญสุด ให้ใช้วิธีอื่นที่ไม่ทำลายความสมดุล เช่น สีพื้นการ์ดต่างออกไป หรือเส้นขอบเน้น — **ห้ามใช้วิธีขยายขนาดตัวเลข**
- คงสไตล์ B และโทนสีเดิม (รายรับเขียว / รายจ่ายชมพู / คงเหลือ)

## 2. แก้ชิป "ตัวอย่างหมวดหมู่" ในหน้าเพิ่ม/แก้หมวดหมู่ — มี 2 ปัญหา

### 2.1 บั๊ก: ชิปตัวอย่างไม่อัปเดตตามที่เลือก

ตารางเลือกไอคอนและวงเลือกสีทำงานถูกต้องแล้ว แต่ **ชิป "ตัวอย่างหมวดหมู่" ด้านบนไม่อัปเดต** — ยังค้างแสดงไอคอนวงกลมเปล่าและค่า default "Circle / coral" ตลอด แม้จะเลือกไอคอน/สีอื่นแล้ว (เป็นทั้งบนจอคอมและมือถือ)

สาเหตุที่น่าจะเป็น: ค่าไอคอน/สีที่ผู้ใช้เลือกไม่ได้ถูกผูก (bind) กับชิปตัวอย่าง — เป็นปัญหาการเชื่อม state

ที่ต้องการ:
- แตะเลือกไอคอนในตาราง → ชิปตัวอย่างเปลี่ยนเป็นไอคอนนั้นทันที
- แตะเลือกสี → ชิปตัวอย่างเปลี่ยนพื้นเป็นสีนั้นทันที
- ข้อความใต้ชิปอัปเดตตามค่าที่เลือกจริง
- ตรวจสอบว่าตอนกด "เพิ่มหมวดหมู่" ค่าไอคอน/สีที่เลือกถูกบันทึกลง database จริง (ฟิลด์ `icon`, `color`)
- โหมดแก้ไข: เปิดหน้าแก้หมวดหมู่เดิม → ชิปตัวอย่าง ตาราง และวงสี ต้องโชว์ค่าปัจจุบันของหมวดนั้นถูกต้อง

### 2.2 แก้ layout ชิปตัวอย่างให้สมดุล

ตอนนี้ในชิปตัวอย่าง ไอคอนถูกจัดไว้กลางกล่อง ส่วนข้อความ ("ตัวอย่างหมวดหมู่ / Circle / coral") ชิดขวาสุด ทำให้มีช่องว่างโล่งกลางกล่อง ดูไม่สมดุล ไม่เป็นชิ้นเดียวกัน

ที่ต้องการ:
- จัดไอคอนกับข้อความให้เป็นกลุ่มเดียวกัน — ไอคอนกับชื่ออยู่ติดกัน (เช่น ไอคอนซ้าย ข้อความต่อทางขวาของไอคอน) เหมือนแถวรายการหมวดหมู่ที่อื่นในแอป
- ไม่มีช่องว่างโล่งกลางกล่อง
- ชิปตัวอย่างควรหน้าตาใกล้เคียงกับแถวหมวดหมู่จริงที่ผู้ใช้จะเห็นในแอป เพื่อให้เป็น "ตัวอย่าง" ที่สมจริง
- คงสไตล์ B

## ทดสอบหลังแก้

- การ์ดสรุป 3 ใบดูเป็นชุดเดียวกัน สมดุล ทั้งบนจอคอมและมือถือ
- เลือกไอคอน/สีต่างๆ → ชิปตัวอย่างเปลี่ยนตามทุกครั้ง
- เพิ่มหมวดใหม่แล้วไอคอน/สีไปแสดงถูกต้องในหน้ารายการหมวดหมู่และหน้าอื่นๆ
- ฟังก์ชันเดิมทั้งหมดยังทำงานครบ
