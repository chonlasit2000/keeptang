# keeptang — แก้งานค้างจาก Phase 1 (ก่อนเริ่ม Phase 2)

> สเปกงานแก้ไขสำหรับ Codex — เก็บงานค้าง 2 จุดจาก Phase 1
> คัดลอกไฟล์นี้ไปวางใน Codex หรือวางไฟล์ไว้ในโฟลเดอร์โปรเจกต์แล้วสั่งให้อ่าน
> แนะนำทำไฟล์นี้ให้จบก่อน แล้วค่อยเริ่ม `keeptang-phase2-spec.md`

**สำคัญ:** ห้ามกระทบฟังก์ชันเดิมที่ทำงานได้แล้ว — แก้เฉพาะ 2 จุดด้านล่าง

---

## 1. เอา numpad ในแอปออก — ใช้คีย์บอร์ดของเครื่องแทน

**ปัจจุบัน:** หน้าเพิ่ม/แก้รายการมีแป้นตัวเลข (numpad) ที่แอปวาดเอง ให้กดป้อนจำนวนเงิน

**ที่ต้องการ:** เอา numpad ที่แอปวาดเองออกทั้งหมด เปลี่ยนช่องจำนวนเงินเป็น input ปกติที่เรียกคีย์บอร์ดของอุปกรณ์ขึ้นมาเอง
- ช่องจำนวนเงินใช้ `<input>` ที่ตั้ง `inputmode="decimal"` — บนมือถือจะเด้งแป้นตัวเลขของเครื่องขึ้นมาให้ บนคอมพิมพ์ด้วยคีย์บอร์ดได้ปกติ
- ยังคงรับเฉพาะตัวเลข (และจุดทศนิยม) — ป้อนติดลบหรือตัวอักษรไม่ได้เหมือนเดิม
- เงื่อนไขเดิมคงไว้: จำนวนเงินต้องมากกว่า 0 ถึงจะบันทึกได้
- ลบโค้ด/คอมโพเนนต์ของ numpad เดิมที่ไม่ใช้แล้วออกให้เรียบร้อย ไม่ทิ้งโค้ดตาย
- ช่องจำนวนเงินยังควรเด่น อ่านง่าย ตามสไตล์ B เหมือนเดิม — เปลี่ยนแค่วิธีป้อน ไม่ใช่ดีไซน์ความเด่นของช่อง

---

## 2. แก้บั๊ก: ส่วนเลือกวันที่หลุดกรอบขวาบนมือถือ

**ปัญหา:** ในหน้าเพิ่ม/แก้รายการ เมื่อเปิดบนมือถือ (จอแคบ) ส่วนเลือกวันที่ล้นออกนอกกรอบด้านขวาของหน้าจอ

**ที่ต้องการ:**
- ส่วนเลือกวันที่ต้องอยู่ในกรอบหน้าจอเสมอบนทุกขนาดจอ ไม่ล้นขอบขวา ไม่ทำให้เลื่อนแนวนอนได้
- ตรวจ container/ความกว้างของ input วันที่ — ตั้งให้ยืดหดตามพื้นที่ (เช่น `w-full`, `max-w-full`, `box-sizing` ถูกต้อง)
- ทดสอบบนจอแคบ (~375px) และจอมือถือจริง — ส่วนวันที่ต้องแสดงครบ กดใช้งานได้สะดวก
- ไม่กระทบการแสดงผลบนจอคอม

---

## ทดสอบหลังแก้

- มือถือ: เปิดหน้าเพิ่มรายการ → ช่องจำนวนเงินเรียกแป้นตัวเลขของเครื่อง, ส่วนวันที่อยู่ในกรอบ ไม่ล้น
- คอม: ช่องจำนวนเงินพิมพ์ได้ปกติ, หน้าเพิ่มรายการแสดงผลปกติ
- เพิ่ม/แก้รายการได้ตามปกติ, จำนวนเงิน 0 ยังบันทึกไม่ได้
- ฟังก์ชันเดิมอื่นๆ ทั้งหมดยังทำงานครบ
