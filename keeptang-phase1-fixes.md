## 1. ทำให้ responsive — รองรับทั้งจอคอมและมือถือ (งานหลัก)

**ปัญหา:** ตอนนี้ UI ออกแบบมาสำหรับมือถือ (คอลัมน์เดียวแคบๆ) พอเปิดบนจอคอมกว้างๆ เนื้อหายังลีบอยู่กลางจอ ใช้พื้นที่ไม่คุ้ม ดูใช้งานยาก

**เป้าหมาย:** โค้ดชุดเดียว แต่ layout ปรับตามขนาดจอ (responsive) — ไม่แยกเป็นสองเว็บ

ใช้ breakpoint ของ Tailwind แบ่งเป็น 2 โหมด — เกณฑ์แบ่งที่ `md` (768px):

**จอเล็ก (มือถือ — ต่ำกว่า 768px):** คงของเดิมไว้ทั้งหมด
- คอลัมน์เดียว
- แถบเมนูล่าง (bottom nav)
- การ์ดสรุป 3 ค่าเรียงต่อกัน

**จอใหญ่ (คอม/แท็บเล็ต — ตั้งแต่ 768px ขึ้นไป):**
- จำกัดความกว้างเนื้อหาสูงสุด (เช่น `max-w-5xl`) จัดกึ่งกลาง ไม่ให้ยืดเต็มจอ 4K จนอ่านยาก
- เปลี่ยน bottom nav เป็น **sidebar ด้านซ้าย** (โลโก้ keeptang + เมนู หน้าแรก/รายการ/ตั้งค่า) — เนื้อหาหลักอยู่ขวา
- การ์ดสรุป 3 ค่า (รายรับ/รายจ่าย/คงเหลือ) เรียงเป็น **แถวแนวนอน 3 คอลัมน์**
- รายการ transaction: ใช้พื้นที่กว้างขึ้น อ่านสบายขึ้น อาจวางเป็น 2 คอลัมน์ได้ถ้าเหมาะ
- ปุ่มลอย (+) มุมขวาล่าง: บนจอใหญ่ย้ายเป็นปุ่มปกติใน sidebar หรือมุมบนขวาก็ได้

**คงสไตล์ B เดิมทุกอย่าง** — สีครีม/coral/พาสเทล, มุมโค้ง, ฟอนต์ เปลี่ยนแค่การจัดวาง ไม่เปลี่ยนธีม
ทดสอบโดยย่อ/ขยายหน้าต่างเบราว์เซอร์ — layout ต้องสลับลื่น ไม่มีของล้นหรือซ้อนทับ

## 2. แก้ bug: ตัวเลขเงินล้นออกนอกการ์ด

**ปัญหา:** ในแถวรายการ ตัวเลขจำนวนเงิน (เช่น `-฿200`, `+฿20,000`) และคำว่า "ลบ" ล้นทะลุขอบการ์ดด้านขวา (เห็นได้ทั้งหน้า Dashboard และหน้ารายการทั้งหมด)

**ต้องแก้:** จัด layout ของแถวรายการให้ทุกองค์ประกอบอยู่ในกรอบการ์ดเสมอ ไม่ว่าตัวเลขจะยาวแค่ไหน
- ใช้ fl/grid ที่จัดสรรพื้นที่ถูกต้อง — ส่วนชื่อหมวด/โน้ตยืดหดได้ ส่วนตัวเลขกว้างคงที่พอ
- ทดสอบกับจำนวนเงินหลักหลายขนาด: `฿50`, `฿3,000`, `฿250,000` — ต้องไม่ล้นทุกกรณี
- ถ้าข้อความหมวด/โน้ตยาวเกิน ให้ตัดด้วย ellipsis (`...`) แทนที่จะดันของอื่นล้น

---

## 3. แก้ bug: เปลี่ยน popup ของเบราว์เซอร์เป็น modal ในแอป

**ปัญหา:** ตอนนี้ตอนยืนยันการลบรายการ และตอนจัดการหมวดหมู่ ใช้ `window.confirm()` / `window.alert()` ของเบราว์เซอร์ ซึ่งหน้าตาไม่เข้ากับแอปและดูแปลกบนมือถือ

**ต้องแก้:** สร้าง modal/dialog component ของแอปเอง ใช้แทน `window.confirm`/`alert` ทุกจุด
- ดีไซน์ตามสไตล์ B — การ์ดขาว มุมโค้ง พื้นหลังมีฉากหลังจางๆ (overlay)
- กล่องยืนยันลบ: มีข้อความถาม + ปุ่ม "ยกเลิก" กับ "ลบ" (ปุ่มลบใช้สีแดงเตือน)
- ใช้ component นี้ซ้ำได้ทั้งแอป (reusable)

---

## 4. ปรับตัวกรองหน้ารายการทั้งหมด: เลือกชนิดก่อน แล้วค่อยเลือกหมวด

**ปัญหา:** ตอนนี้ตัวกรองหมวดหมู่รวมทั้งรายรับและรายจ่ายปนกันใน dropdown เดียว เลือกยาก

**ต้องแก้:** ทำเป็น 2 ขั้น
- เพิ่มปุ่มสลับ (segmented control) ด้านบน: **ทั้งหมด / รายรับ / รายจ่าย**
- dropdown หมวดหมู่ด้านล่าง แสดงเฉพาะหมวดของชนิดที่เลือกไว้ (เลือก "รายจ่าย" → dropdown โชว์เฉพาะหมวดรายจ่าย)
- เลือก "ทั้งหมด" → dropdown แสดงทุกหมวดเหมือนเดิม

---

## 5. ปรับหน้าจัดการหมวดหมู่ให้แสดงไอคอนและสี

**ปัญหา:** หน้าตั้งค่า > จัดการหมวดหมู่ แสดงเป็นข้อความล้วน ไม่มีไอคอนหรือสี ดูยาก แยกแต่ละหมวดลำบาก

**ต้องแก้:** แต่ละแถวหมวดหมู่ให้แสดงชิปไอคอนพร้อมสีจริง
- ข้อมูล `icon` และ `color` มีอยู่ใน database (ตาราง `categories`) อยู่แล้ว — แค่ดึงมาแสดง
- รูปแบบแถว: ชิปไอคอนสีพาสเทล (เหมือนในหน้า Dashboard) + ชื่อหมวด + ปุ่มแก้/ลบ
- ตอนเพิ่ม/แก้หมวดหมู่ ให้เลือกไอคอนและสีได้ด้วย (ถ้ายังไม่มี)

---

## หมายเหตุ: ฟิลด์ `grp` ในตาราง categories

ฟิลด์ `grp` (กลุ่ม: need/want/saving/reward) **ให้คงไว้ ห้ามลบ** — Phase 1 ยังไม่ได้ใช้แสดงผล
แต่ Phase 2 จะใช้ทำผังกระแสเงินสด (Flow) แบ่งว่าเงินไปกลุ่มไหน จึงเก็บไว้ในฐานข้อมูลตั้งแต่ต้น

---

## ลำดับงานที่แนะนำ

1. ข้อ 2, 3, 4, 5 (บั๊ก UI ย่อย — แก้ก่อนเพราะกระทบจุดเดียวๆ)
2. ข้อ 1 (responsive — งานใหญ่ แตะหลายไฟล์ ทำทีหลังจะได้ไม่ชนกับข้ออื่น)
3. ทดสอบทุกข้อ: ย่อ/ขยายหน้าต่างเบราว์เซอร์, เปิดบนมือถือจริง
4. ยืนยันว่าฟังก์ชันเดิม (บัญชี/RLS/คำนวณยอด/เพิ่ม-แก้-ลบ) ยังทำงานครบเหมือนเดิม
